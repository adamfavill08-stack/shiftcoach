# Play Store: Closed Test Required Before Production

## ⚠️ Important: Google Play Now Requires Closed Test

Google Play Console now **requires** you to run a **Closed Test** before you can apply for Production access.

---

## Required Steps (New Process)

### Step 1: Create Closed Test (Required)
1. **Play Console → Your App → Testing → Closed testing**
2. **Create new release**
3. Upload your AAB file
4. **Add testers** (at least yourself)
5. **Save** and **Review release**
6. **Start rollout to Closed testing**

### Step 2: Test for Required Period
- Google requires the closed test to meet certain criteria
- Usually need to have some testers and activity
- Minimum: 1-2 days with at least 1 tester

### Step 3: Apply for Production Access
1. After closed test is running, go to **Production**
2. Click **"Apply for access to production"**
3. Answer questions about your closed test:
   - How many testers?
   - How long did you test?
   - What did you test?
   - Any issues found?
4. Submit application

### Step 4: Wait for Approval
- Google reviews your application
- Usually approved within 1-3 days
- You'll get an email when approved

### Step 5: Release to Production
- Once approved, you can create releases in Production
- Upload AAB
- Submit for review

---

## Fastest Path (Still Pretty Fast)

**Total time: 3-5 days**
1. **Day 1**: Create closed test, add yourself as tester
2. **Day 1-2**: Test on your phone
3. **Day 2-3**: Apply for production access
4. **Day 3-5**: Wait for approval
5. **Day 5+**: Release to production

---

## How to Set Up Closed Test (5 minutes)

1. **Play Console → Testing → Closed testing**
2. **Create new release**
3. Upload your AAB file
4. **Testers → Create email list**
5. Add your email (and anyone else)
6. **Save**
7. **Review release → Start rollout**

**Minimum requirements:**
- At least 1 tester (yourself is fine)
- Test for at least 1-2 days
- Actually test the app (download and use it)

---

## Questions You'll Need to Answer

When applying for production, Google will ask:
- How many testers did you have?
- How long did you test?
- What features did you test?
- Did you find any issues?
- How did you address issues?

**Answers:**
- Testers: "1-5 testers" (yourself + friends/family)
- Duration: "2-3 days"
- Features: "All core features - authentication, subscriptions, sleep tracking, etc."
- Issues: "No critical issues found" (or list any you fixed)
- Addressing: "Fixed before production release"

---

## Tips for Fast Approval

1. **Actually test the app** - Don't just upload and wait
2. **Add real testers** - Even just 2-3 people helps
3. **Test for 2-3 days minimum** - Shows you're serious
4. **Answer questions honestly** - Don't lie about testers/duration
5. **Test core features** - Authentication, payments, main features
